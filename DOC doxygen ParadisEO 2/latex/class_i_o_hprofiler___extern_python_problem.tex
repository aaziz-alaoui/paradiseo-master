\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Input\+Type$>$\newline
class I\+O\+Hprofiler\+\_\+\+Extern\+Python\+Problem$<$ Input\+Type $>$}

Classes for calling problems written in Python. 

...

\begin{DoxyAuthor}{Author}
Johann Dreo A proxy for problems written in the Python programming language.
\end{DoxyAuthor}
Given a Python module in which exists an instance of a class honoring the needed interface, this class be be use as a proxy that will call the Python code of a problem.


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{class_i_o_hprofiler___extern_python_problem}{IOHprofiler\_ExternPythonProblem<int>}} pypb(\textcolor{stringliteral}{"mymodule"}, \textcolor{stringliteral}{"myinstance"});}
\DoxyCodeLine{\textcolor{keywordtype}{size\_t} d = pypb.IOHprofiler\_get\_number\_of\_variables();}
\DoxyCodeLine{std::vector<int> sol(d,6);}
\DoxyCodeLine{\textcolor{keywordtype}{double} fit = pypb.evaluate(sol);}
\end{DoxyCode}


\begin{DoxyNote}{Note}
Given that this class operates on a class instance a given Python module may host several problems. However, you still need to create one C++ class instance per Python class instance.
\end{DoxyNote}
The interface is formed by several member functions, which should be implemented by the targeted Python class.

An abstract base class enforcing the default interface is distributed with I\+OH in the {\ttfamily ioh.\+py} file. Users may inherit from this class to ensure that they correctly implemented the interface, albeit this is not mandatory (given that Python does not enforce type checking).

Default names of the members can be changed by modifying the values of the {\ttfamily $\ast$api$\ast$} parameters passed to this class\textquotesingle{} constructor.

All the {\ttfamily get\+\_\+$\ast$} member functions of the Python interface just return a parameter. Only the {\ttfamily call} function takes arguments (i.\+e. a candidate solution to the problem). This class send a call tuple to the Python function, so that you can either define a fixed number of explicit variables, either capture the tuple as a positional argument expansion\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\# Example for variable dimensions:}}
\DoxyCodeLine{def call(\textcolor{keyword}{self}, *args ):}
\DoxyCodeLine{    return sum(*args)}
\DoxyCodeLine{}
\DoxyCodeLine{\# Example for fixed dimensions:}
\DoxyCodeLine{def call(self, x0, x1, x2):}
\DoxyCodeLine{    return x0+x1+x2}
\end{DoxyCode}
